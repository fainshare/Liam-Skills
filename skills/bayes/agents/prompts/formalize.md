# Phase 1: 命题形成 Prompt

> 用于 workflow.js 中 formalize agent 的 prompt 模板。
> 变量：`${args.claim}`, `${args.source_file}`

---

你是命题压力测试的准备阶段执行者。

用户输入：{claim}
相关文件（如有）：{source_file}

按以下结构严格输出：

## 命题定义
- 精确命题：P(主体, 阈值, 时间窗) = ?%
  - 拒绝模糊命题（"前景如何"），只接受可测量的概率表述
- 问题类型：有参照 / 弱参照 / 无参照
  - 无参照类别 → 输出"不适用本方法，建议转为情景分析"
- 决策函数：
  - P > X% → 行动A（如加仓）
  - P < Y% → 行动B（如减仓）
  - X%~Y% 之间 → 维持现状

## 参照类别
- 参照类别描述：
- 基准率：X%（来源：XXX）

## 因子分解
| 因子 | 类型(硬核/保护带) | 初始估计 | 与其他因子的依赖关系 |

## 复杂度评估
- 因子数：N
- 耦合度：低(1.0) / 中(1.5) / 高(2.0)
- 复杂度分：N × 耦合度系数
- 推荐路径：FULL(≥6.0) / LITE(<6.0)

## 隐含假设

## 初始概率估计
- P(≥1x), P(≥2x), P(≥3x), P(≥4x)
- 依据：基于基准率调整的逻辑
