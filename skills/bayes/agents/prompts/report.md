# Phase 5: 最终报告生成 Prompt

> 使用 templates/report-template.md 的七章节结构。

---

生成最终命题压力测试报告。

七章节结构：

## 一、命题定义与决策函数
（从 Phase 1 提取命题、决策边界、参照类别）

## 二、对抗审查记录
- 每轮：模式（FULL/LITE）、审查员数、物质性缺陷清单
- 方向偏斜统计 + Lakatos 判据
- 退化警报（如有触发）

## 三、溯源与修正记录
- 每个被溯源缺陷：一手源证据 + 层级 + 修正结论

## 四、条件概率树
- 因子 → 条件概率链 → 联合概率
- 终端节点概率之和 ≈ 100%

## 五、多阈值累积概率分布
| 回报阈值 | 概率 | 误差带 | 核心前提 |
- E[return], P(亏损), 条件 E(亏损时), 条件 E(盈利时)

## 六、修正轨迹
| 轮次 | P(≥1x) | P(≥2x) | P(≥3x) | P(≥4x) | 主要修正原因 |

## 七、停止判断
- 最终判决 + 依据
- ✅ SATURATED / ⚠️ BUDGET_STOP

### 方法论局限声明（必填）
- prompt 级独立性 ≠ 认知级独立性
- 误差带由 LLM 自评，±5pp 下限是工程防护
- 单命题评估，不替代跨标的比较决策
- 无法检测审查者与被审查者共享的系统性盲区
