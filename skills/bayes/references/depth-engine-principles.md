# 深度引擎设计原则：v4.0 北极星

> 本文档定义 bayes skill v4.0 的核心生成机制——如何产生超越初始判断的认知增量。
> 不替代 `theoretical-foundations.md`（那个讲认识论纪律），本文档讲**生成机制**。
>
> *版本：1.2.0 | 日期：2026-06-06 | 来源：三视角诊断（Gödel/Darwin/Feynman）+ 三轮费曼审查 + 杨振宁直觉修正方法论 + 软证据层突破*

---

## 一、问题：为什么 v3.4 不产生 alpha

v3.4 的循环（对抗审查 → 溯源 → 概率修正）经三视角独立诊断，结论一致：

- **Gödel**：封闭公理系统只能到达其公理可达的定理。Phase 1 锁定命题后，后续循环只能在固定框架内微调——永远无法发现"问题本身问错了"。
- **Darwin**：8 个预定义审查维度 = 固定基因组。没有突变机制，系统只做稳定化选择（收敛到均值），不产生相变。
- **Feynman**：去掉术语后，v3.4 是"让同一个 LLM 多想几遍然后取最后一个答案"。信息增量在第 2 轮后趋近于零。

**核心矛盾一句话**：v3.4 的循环是"自我批评 → 自我确认"。没有外部事实冲击，系统在自己的认知边界内打转，永远收敛到 LLM 训练分布的众数——而众数恰恰是市场共识 = 零 alpha 区域。

## 二、核心命题：深度 = 惊讶驱动的层次跃迁

### 形式定义

$$\text{深度} \neq \text{迭代轮次}$$

$$\text{深度} = \text{模型预测被外部事实击穿的次数} \times \text{跃迁成功率}$$

每次"击穿"= 一次层次跃迁的**机会**（不是保证）。跃迁是否成功取决于人类能否基于惊讶重构认知。

### 层次定义

| Level | 名称 | 特征 | 举例（康方生物） |
|-------|------|------|---------------|
| 0 | 现象描述 | 可直接观察的事实 | "当前 PE 35x，管线 20+ 品种" |
| 1 | 模式识别 | 可统计的规律 | "创新药从亏转盈的估值重估通常 2-3x" |
| 2 | 生成机制 | 为什么这个模式存在 | "市场对 PD-1 后管线定价缺失源于信息不对称+临床长周期" |
| 3 | 结构约束 | 机制的边界条件 | "中国医保支付体系的天花板由 DRG 覆盖率决定，非药品本身" |

### 与 v3.4 的对比

| 维度 | v3.4 | v4.0 |
|------|------|------|
| 深度指标 | 误差带收敛（衡量稳定性） | 惊讶事件数 × 跃迁率（衡量认知增量） |
| 深入动力 | LLM 自我挑战（无外部冲击） | 数据击穿预测（不可被 LLM 绕过） |
| 停止条件 | "带宽在决策区间内" | "三维度预测全部验证通过"或"人类判断已够" |
| 框架可变性 | Phase 1 锁死 | 人类可基于惊讶触发 reframe |

## 三、三层主从架构

### 因果链

```
命题 → 提取预测（方向性 + 前提验证型）
       │
       ├→ B1 硬验证（API 数值）→ 二值：对/错
       │
       ├→ B2 软验证（双读对抗）→ 三值：一致同向/一致反向/分歧
       │
       └→ 不可测 → 人类确认模式
              │
              ▼
         覆盖率检查（D7）
         ├→ >50% 降级 → ⚠️ "无惊讶≠安全" 警告
         └→ ≤50% 降级 → 继续
              │
              ▼
         惊讶汇总（硬惊讶 + 软惊讶 + 解读分歧）
              │
       ┌──────┴──────────────────────────────────┐
       │ 无惊讶                                    │ 有惊讶
       ▼                                          ▼
  验证通过                                  具名框架解释
  （该维度已确认）                            ≥2 竞争性归因
                                            + 新可测预测
                                                  │
                                                  ▼
                                             人类决策
                                      ┌───┬────┼────┬───┐
                                      │   │    │    │   │
                                   微调 跃迁  补数据  暂存
                                  (Level  │   (回到   (存入档案
                                   不变)  │   引擎)   top-5/90天)
                                          ▼
                                    人类重述（建议）
                                    (用自己的话表述新命题)
                                          ▼
                                    Phase 1 更新
                                     (Level +1)
```

### 核心引擎：预测-测量-惊讶循环

这是系统的心脏。它不依赖 LLM 的"意志"——LLM 做预测提取，但验证和惊讶判定由代码执行。

| Step | 做什么 | 执行者 | LLM 能绕过？ |
|------|--------|--------|:-:|
| A | 从命题提取三维度方向性预测（含反面条件） | LLM | — |
| B1 | 硬验证：用白名单 API 验证预测方向（数值型） | 代码 + API | ✗ |
| B2 | 软验证：用 deep-research 获取非结构化源，双独立 LLM 解读方向 | 代码获取 + 双 LLM 解读 | 源选择 ✗ / 解读有对抗 |
| C | 预测方向 vs 实测方向，二值判定（硬）或三值判定（软：一致/反向/分歧） | 代码 | ✗ |

#### 三维度独立信息通道

三个维度的信息来自不同的生产过程，具有结构性独立——不是"不同 prompt"的伪独立。

| 维度 | 验证什么 | 数据源白名单 | 可查指标举例 |
|------|---------|-------------|-------------|
| 宏观 | 行业/政策/周期方向 | FRED、各国央行报告、BIS、国家统计局、海关总署 | 利率方向、PMI、行业产出增速 |
| 公司 | 财务/运营/管线方向 | SEC Filing (10-K/10-Q/20-F)、港交所/A股公告、IR Presentation | 毛利率方向、收入增速、管线里程碑 |
| 市场 | 价格/资金/波动率方向 | Bloomberg、期权链、融券余额、北向资金 | 股价 vs 分析、IV vs HV、做空比例变化 |

#### 预测质量约束：区分力校验

每个预测必须同时附带正面条件和反面条件，形成可判定的假设检验：

```
预测结构 = {
  factor: "管线进展",
  if_thesis_correct: "2026H2 至少 1 个 III 期数据读出 → 方向：正面催化",
  if_thesis_wrong: "2026H2 无 III 期数据 OR 数据失败 → 方向：负面信号",
  data_source: "公司公告 + ClinicalTrials.gov",
  verifiable_by: "2026-12-31"
}
```

**硬约束**：`if_thesis_correct` 的方向 ≠ `if_thesis_wrong` 的方向，否则该预测被拒绝（无区分力）。

#### 前提验证型预测（杨振宁第8条启发）

> "永远不要把所谓'不验自明'的定律视为是必然的。" — 杨振宁

除了验证命题的推论，还应验证**命题依赖的"不验自明"前提**本身是否成立。这是杨振宁和李政道发现宇称不守恒的方法论内核：所有人直觉认为宇称守恒 → 他们问"有人实际证明过吗？" → 答案是没有 → 诺贝尔奖。

```
前提验证预测 = {
  type: "premise_audit",
  premise: "中国创新药企的海外授权能力持续提升",
  evidence_required: "实际已完成的 license-out 交易数量和金额趋势",
  verified_by: "SEC/港交所公告中的实际交易记录",
  last_verified: null,   // 从未验证 = 高优先级
  status: "unverified"   // unverified | confirmed | refuted
}
```

**操作化**：Phase 1 命题形式化时，系统要求列出 3-5 个"本命题默认为真但从未验证"的前提。这些前提进入预测池，与常规预测一起接受数据验证。

**前提 refuted = 最高优先级惊讶**——因为前提被击穿时，整个命题的逻辑地基动摇，这是最可能触发 Level 跃迁的信号。

#### 软证据层：双读对抗机制（v1.2 新增）

对非结构化信息源（公告全文、研报、新闻、访谈纪要、专利），D3 的原始表述（"验证不经过 LLM"）过于严格，导致 ~70% 的因子被放弃。v1.2 的突破：**拆解 D3 的真正守护目标——切断确认偏误的传染链，而非禁止一切 LLM 参与。**

确认偏误的传染链：`LLM 选源 → LLM 解读 → LLM 判断`（全链同一主体 = v3.4 的病因）

切断方式：**源选择由代码控制**（白名单域名 + 预定义查询模板），LLM 不决定"看什么"。

```
源获取（代码控制）→ 原文 → Agent A 提取方向 ─┐
                                              ├→ 比对
                    → Agent B 提取方向 ─┘
```

三种结果及系统行为：

| 结果 | 含义 | 系统行为 | 证据标记 |
|------|------|---------|---------|
| 双方一致 + 与预测同向 | 软确认 | 该维度通过 | `soft_confirmed` |
| 双方一致 + 与预测反向 | **软惊讶** | 触发解释层 | `soft_surprise` |
| 双方分歧 | **解读惊讶** | 原文 + 两种解读呈现给人类 | `interpretation_divergence` |

**关键洞察**：第三种结果（分歧）的投资价值最高——市场未定价某信息，往往恰恰是因为其方向性 genuinely ambiguous。发现模糊点 = 发现潜在 alpha 区域。

**软证据层的约束**：
1. 源文档原文必须可追溯（人类可查验原始 URL/文件）
2. Agent A 和 Agent B 使用不同基座模型或不同 system prompt（见下方"独立性策略"）
3. 呈现时硬证据（API 数值）与软证据（文本解读）视觉区分，不混为一谈
4. 软惊讶的权重低于硬惊讶——不能仅凭软证据触发 Level 跃迁建议
5. **源质量预筛**：基于结构化信号检测（非字符数——中文信息密度极高，"董事长辞职"5字符即可触发跃迁）。三选二通过即为有效源，否则标记 `low_signal`：
   - **实体锚定**：含具体公司名/股票代码/人名
   - **动作谓词**：含决策动词（终止/批准/辞任/收购/发行/暂停/获批/延期）
   - **源权威性**：来自官方披露渠道（交易所/监管机构/公司 IR/央行）
   
   `low_signal` 源的分歧不计入 alpha 候选，仅标注"信息稀疏"

**独立性策略（按环境能力分级）**：

| 级别 | 条件 | Agent A / Agent B 配置 | 真实独立性 |
|------|------|----------------------|-----------|
| L1（降级） | 仅有 Claude | 保守 prompt / 激进 prompt | 低：同模型同权重，仅 prompt bias 差 |
| **L2（当前实现）** | Claude + Qwen3.7-Max | **Agent A = Claude（workflow 原生）/ Agent B = Qwen3.7-Max（Idealab API）** | 中-高：不同架构 + 不同训练语料（尤其中文域） |
| L3（扩展） | Claude + Qwen + Gemini | 三模型多数投票 | 高：结构性独立，盲区交集最小化 |

**L2 为当前实现方案**：Agent B 优先使用 Qwen3.7-Max（通过 Idealab OpenAI 兼容接口调用），与 Claude 在架构和训练数据上完全独立——尤其在中文财经领域，Qwen 的中文语料覆盖与 Claude 存在结构性差异，这正是双读对抗需要的。同模型双 prompt（L1）仅在外部 API 不可用时作为降级方案。

**覆盖率分层报告**（禁止用聚合数字制造虚假安全感）：

| 层 | 覆盖率 | 置信度 | 说明 |
|---|--------|--------|------|
| 硬覆盖 | ~30% | 高（API 数值，代码判定） | 等同于"GDP +3.2%"级别的事实 |
| 软覆盖 | ~25-35% | 中（有效置信折扣 ~60%） | 双 LLM 一致解读 ≠ 事实为真；管理层公告有美化 incentive |
| 未覆盖 | ~35-45% | 无（纯人类判断区） | D7 覆盖率警告在此生效 |

**系统在报告中必须分层呈现，不得合并为"覆盖率 55-65%"**——聚合数字把高置信和低置信混为一谈，比坦承"70% 不可测"更危险。

#### 不可测因子降级规则

投资命题中约 30% 的因子可通过 API 验证。对不可测因子：

1. 系统标记为"不可自动验证"
2. 向人类提问："根据你的信息，[因子 X] 的当前方向是...？"
3. 人类回答视同数据输入，进入惊讶判定

不可测 ≠ 不重要。降级保证了系统不在无数据的维度上"假装验证"。

### 解释层：具名框架对抗

**仅当惊讶发生时激活。** 输入是一个具体事实："预测方向 A 但实测方向 B"。

#### 为什么用"具名框架"而非"多模型"

| 维度 | 匿名多 agent (v3.4) | 具名框架 (v4.0) |
|------|-------------------|---------------|
| 独立性来源 | prompt 差异（伪独立） | 哲学立场差异（外部可验证） |
| 可审计性 | 无法验证 reviewer 的"角色扮演"质量 | 可查该框架的已知立场 |
| 覆盖空间 | LLM 训练分布内重组 | 已被验证的投资哲学集合 |
| 新变量来源 | LLM 编造 | 各框架的核心关注维度（已知且固定） |

#### 框架池（非穷尽，可按命题类型选取 3-4 个）

| 框架 | 核心关注 | 典型解释方向 | 代表文献 |
|------|---------|-------------|---------|
| 价值投资 | 安全边际、内在价值 | "市场暂时错误定价" | Graham *Security Analysis*; Greenwald *Value Investing* |
| 反脆弱 | 凸性、尾部风险、非线性 | "你低估了左尾/高估了线性外推" | Taleb *Antifragile*; *Statistical Consequences of Fat Tails* |
| 反身性 | 预期与现实的双向反馈 | "价格本身改变了基本面" | Soros *The Alchemy of Finance* |
| 行为金融 | 系统性认知偏差 | "市场参与者集体锚定/过度反应" | Kahneman *Thinking Fast and Slow*; Shiller *Irrational Exuberance* |
| 竞争战略 | 护城河、进入壁垒 | "你错判了竞争格局的稳定性" | Porter *Competitive Strategy*; Greenwald *Competition Demystified* |
| 周期理论 | 均值回归、信用扩张 | "你忽略了周期位置" | Dalio *Principles for Navigating Big Debt Crises*; Howard Marks *Mastering the Market Cycle* |

#### 输出结构

每个框架的解释必须：
1. 明确归因："从 [框架名] 视角看，Y 而非 X 是因为..."
2. 附带该解释隐含的新可测预测："如果这个解释正确，那么 Z 应该→ 方向 C"
3. 新预测可喂回核心引擎的下一轮验证

### 决策层：人类判断

系统呈现三件事，人类做一个选择：

**呈现**：
1. 惊讶事实（具体的预测 vs 实测）
2. 具名框架解释（2-4 个竞争性归因，各附新预测）
3. 当前 Level 标注（基于已发生的跃迁次数）

**人类选择**：
- **(a) 微调参数**：惊讶不改变核心逻辑，调整具体数字。Level 不变。
- **(b) 接受替代假设**：某个框架的解释揭示了之前未建模的因子。触发 Phase 1 因子集修改（Level +1）。
- **(c) 补更多数据**：当前信息不足以判断，指定需要查什么。回到核心引擎。
- **(d) 暂存惊讶**：当前无法判断，但惊讶事实值得记录。存入惊讶档案，下次运行时作为历史输入。

**D5 硬约束**：系统不自动执行 (b)。框架跃迁是不可逆决策（改变后续所有推理方向），必须由人类拍板。

#### 重述步骤：内化机制（杨振宁"新直觉"启发）

> "把新学的知识变成增加了的直觉。" — 杨振宁

当人类选择 (b) 跃迁时，系统不直接修改 Phase 1 因子集，而是要求人类**用自己的话重新陈述命题**：

1. 系统提示："你接受了 [框架X] 的解释。请用一句话重新表述你现在的核心判断。"
2. 人类重述（例："康方的核心逻辑不是管线数量而是支付端的结构性变化"）
3. 系统基于重述更新 Phase 1 的 `precise_statement` 和因子集

**为什么不能跳过**：杨振宁的圆周运动故事的关键不是"知道了向心加速度"，而是"用向量的概念重新理解了运动"。被动接受一个解释 ≠ 真正的认知跃迁。重述迫使系统二加工，完成知识蒸馏（系统二 → 系统一）。

如果人类无法用自己的话重述——这本身就是一个信号：跃迁可能是虚假的，应降级为 (a) 或 (d)。

## 四、七条设计原则

| # | 原则 | 操作化定义 | 类型 | 违反时系统行为 | 理论依据 |
|---|------|-----------|------|-------------|---------|
| D1 | 惊讶是唯一燃料 | 无数据惊讶 → 无深入理由 → 停止或切换维度 | 硬 | 代码阻断进入解释层 | Popper：只有反驳尝试产生知识增长 |
| D2 | 预测必须有区分力 | 正面条件方向 ≠ 反面条件方向；含前提验证型预测 | 硬 | 代码校验拒绝该预测 | Tetlock + 杨振宁第8条："不验自明"非必然 |
| D3 | 源选择不经过 LLM | 数据源由白名单代码控制；解读可用双 LLM 对抗但源文档必须可追溯 | 硬 | 架构隔离源选择；软验证要求双读一致性 | 费曼审查 v2：确认偏误的根在"谁决定看什么"，不在"谁读内容" |
| D4 | 解释用具名框架 | 引用外部可验证的投资哲学立场 | 软 | prompt 引导，不做硬门控 | Delphi 改进：具名 > 匿名 |
| D5 | 跃迁由人类决策+重述 | 系统呈现信号，人类拍板；建议（非强制）用自己的话重述新命题 | 软 | 等待人类输入才能修改 Phase 1；重述为 UX 建议 | Argyris 双环学习 + 杨振宁"新直觉内化"（心理学价值，非可执行约束） |
| D6 | 惊讶可持久化 | 同标的 `surprise-archive.json` 中 `status=unresolved` 条目，按 `created_at` 降序取 top-5 注入 Phase 1；超过 90 天未触发标记 `stale` | 硬 | 系统自动加载 + 自动过期 | 杨振宁"渗透学法" + 噪信比控制 |
| D7 | 低覆盖率不沉默 | 降级为人类确认的预测数 / 总预测数 > 50% 时，输出覆盖率警告（50% 为初始阈值，需回测校准） | 硬 | 禁止输出"无惊讶=安全"结论；建议对不可测维度执行定向搜索 | 费曼审查 v3：D1 反面盲区——"路灯下找钥匙"风险 |

### 原则间的互锁关系

```
D2（区分力+前提审计）保证 D1（惊讶）是真实的且触及地基
D3（源选择隔离+双读对抗）保证 D1 不被 LLM 绕过，同时覆盖率从 30% → 55-65%
D4（具名框架）保证解释层有外部锚点
D5（人类决策+重述建议）保证跃迁经过人类深思；重述促进但不强制内化
D6（持久化+检索策略+过期机制）保证认知积累跨越会话，且噪音不淹没信号
D7（覆盖率警告）保证 D1 的沉默不被误读为安全——低覆盖率时系统主动声明盲区
```

## 七、边界诚实声明

### 能做到

- 纪律化地防止选择性忽视不利数据（核心引擎的硬验证）
- 用具名框架扩展解释空间（避免单一视角独断）
- 提供结构化的"框架跃迁机会"（惊讶 → 解释 → 人类决策）
- 产生"数据驱动的框架修正型 alpha"

### 不能做到

| 边界 | 原因 | 缓解措施 |
|------|------|---------|
| 不可测因子占 ~35-45%（v1.2 后） | 管理层能力/隐性竞争壁垒无任何信息源 | 降级为人类确认模式 + D7 覆盖率警告 |
| 软证据解读不完美 | 双 LLM 可能共享系统性盲区 | 分歧标记 + 原文可追溯 + 人类终审 |
| 不产生纯创造型 alpha | 需要训练分布外的假设 | 具名框架提供有限的视角扩展 |
| 不克服人类自身的框架锁定 | 系统呈现信号 ≠ 人类能解读 | 多框架解释 + 解读分歧呈现降低锁定 |
| 具名框架集有限 | 投资哲学是已知有限集 | 框架池可扩展，但不可能穷尽 |
| 硬证据维度往往是零 alpha 区域 | 市场效率对已量化信息定价更快 | 软证据层覆盖"已存在但未结构化"的信息 = 效率缝隙 |

**诚实的一句话**：此系统是 alpha 机会的**探测器 + 纠偏器**——它主动从结构化和非结构化信息源中发现与你预期不符的事实，用具名框架呈现你可能没想到的解释。但它不是 alpha **发生器**——最终的跃迁判断和创造性洞见仍然是人类的工作。v1.2 的进攻性来自软证据层：覆盖率从 30% → 55-65%，把"路灯外"的区域也纳入探测范围。

## 八、验证标准

v4.0 实现后的成功/失败判定：

| 标准 | 度量方法 | 最低阈值 |
|------|---------|---------|
| 惊讶产出率 | 三维度中至少 1 个预测方向被击穿 | >0（系统能检测到惊讶） |
| 框架跃迁率 | 人类选择 (b) 的比例 | >0（至少有机会跃迁） |
| 认知增量 | 报告中"事前不知道的事实"数量（人类自评） | ≥1 |
| 对照增量 | 同一命题 v3.4 vs v4.0 报告比较 | v4.0 含更多外部事实 |
| 假阳性控制 | 对已知结论正确的命题，不制造虚假惊讶 | 假阳性率 < 30% |

### 失败信号（需要回到设计层修正）

- 系统持续在可测维度制造噪音但核心维度沉默 → 预测提取质量问题
- 具名框架解释高度雷同 → 框架池选取不当
- 人类永远选 (a) 微调 → 系统未能呈现有说服力的替代解释
- 人类永远选 (c) 补数据 → 预测设计脱离当前可验证范围

## 附录 B：杨振宁直觉修正方法论（v1.1 新增理论来源）

> 来源：杨振宁《我的学习与研究经历》（2012，《物理》41卷1期）+ 国科大演讲全文

### 核心认知框架

```
旧直觉（先验）→ 与事实/形式知识冲突（惊讶）→ "必须想清楚，不能放弃"（深度加工）
                                                         ↓
                                              新直觉（后验，成为新的起点）
```

### 对本设计的三条贡献

| # | 杨振宁原则 | 对应修正 | 影响的原则 |
|---|-----------|---------|-----------|
| 1 | "不验自明"非必然（十诫第8条） | 前提验证型预测 | D2 增强 |
| 2 | "把新知识变成增加了的直觉" | 跃迁后重述步骤（内化） | D5 增强 |
| 3 | 渗透学法（容忍模糊，跨时间理解） | 惊讶持久化档案 | D6 新增 |

### 与贝叶斯更新的同构关系

| 杨振宁术语 | 贝叶斯术语 | 本系统对应 |
|-----------|-----------|-----------|
| 旧直觉 | 先验分布 | Phase 1 命题 + 初始概率 |
| 直觉与事实冲突 | 高信息量观测（惊讶度 = -log P(data|prior)） | 核心引擎的惊讶判定 |
| "必须想清楚" | 贝叶斯推断的计算代价 | 解释层 + 人类深度加工 |
| 新直觉 | 后验分布 | 人类重述后的新命题 |
| 渗透学法 | 增量更新 / 延迟融合 | D6 惊讶持久化 |

### 关键引文

> "每一个人都有很多直觉，而直觉有许多是需要修正的。如果你随时能够接受修正直觉的话，就继续在向前进。"

> "直觉与书本知识冲突是最好的学习机会，必须抓住这个机会。"

> "永远不要把所谓'不验自明'的定律视为是必然的。"
