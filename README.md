# 太虚五转：我给 AI 搭了条认知流水线


今天，数字生命卡兹克写了一篇文章《分享2个Vibe Coding必备的超实用Prompt》，讲了第一性原理和对抗式审查。

我在下面留了条留言，介绍了我自己总结的一套方法论 Skill 包，太虚五转。没想到大家很喜欢，希望我展开讲讲。

所以就有了这篇文章。

卡老师的两个 Prompt 确实厉害，一个管生成，一个管验证，首尾呼应。但以我这一年多的 AI 使用经验来看，这个回路中间还缺了好几环。

第一性原理帮你找到好的方案，但方案从「想到」到「靠谱」之间，还有很长的路。对抗式审查帮你找 BUG，但审完之后呢？被否掉的东西里有没有被误杀的？这些都需要引入新的工具来解决。

于是就有了今天这套方法论。

我这套方法论的底层，是两个核心框架。

第一个框架的出处，是孟岩老师的播客《无人知晓》E45，那期叫《人何以自处》，嘉宾是李继刚老师。他分享了自己的思考框架：「万有理论 + 贝叶斯公式 + 奥卡姆剃刀法则」。说它是元认知、是解决一切问题的法门，都不为过。李老师还有个判断我很喜欢：「工业革命拿走了体力，AI 正在拿走脑力。留给人的，是心力。」

这个框架的隐含假设是：世界是可知的、收敛的。贝叶斯终会逼近真相，奥卡姆会选出最简洁的正确解释。确定性问题（比如代码、数学），这个假设成立，用这套足够了。

但万有理论是个理想化的猜想，物理学至今也没找到。而万物都有第一性原理，是可以一层层往下拆的。所以我把它修正成了现在这个版本：「第一性原理 + 贝叶斯公式 + 奥卡姆剃刀法则」。

修正完之后我发现一个问题：投资、社会学、历史人文这些非确定性问题，不能盲目套用。你没法用第一性原理算出市场的「正确定价」，因为市场会因为你的观察而改变。你也没法用奥卡姆剃刀砍掉人性的非理性，因为非理性本身就是系统的一部分。

所以我又延伸出了第二个框架：「有界贝叶斯 + 相变识别 + 时间成本最优」，每一项都是对第一个框架的辩证修正：

- **有界贝叶斯**，对应贝叶斯公式。贝叶斯假定先验明确、新信息不断逼近真相。但非平稳系统里，今天的规律明天可能失效。所以贝叶斯是有界的，你更新不到「真相」，只能更新到「当前最优」。
- **相变识别**，对应奥卡姆剃刀。奥卡姆说「如无必要，勿增实体」。但复杂系统最危险的地方是：最简洁的模型一直正确工作，直到某个临界点突然全部失效。相变识别不找最简解释，找的是「规则本身发生改变」的那个拐点。
- **时间成本最优**，对应第一性原理。确定性问题追求「找到正确答案」，非确定性问题追求「在有限时间内做出最优决策」。你永远不会拥有完整信息，与其等到想清楚再动，不如在行动中迭代。

这里不得不说一下反脆弱这套方法论，它跟第二个框架是异曲同工，底层逻辑完全契合。

塔勒布讲的肥尾效应，就是相变识别的另一面。相变识别找的是「规则发生改变」的拐点，肥尾效应说的是：极端事件发生的频率远超你的预期，而正是这些极端事件改变了规则。2008 年金融危机就是一次肥尾，也是一次相变。平时一切正常，正态分布告诉你尾部风险可以忽略，然后一次极端事件就把规则全改了。平稳期的规律在转折期会失效，这是相变识别的判断，也是肥尾效应的结论。

有界贝叶斯对应反脆弱对预测的态度：既然你没法预测，就别把力气花在预测上，花在构建能从冲击中获益的系统上。时间成本最优对应杠铃策略：大部分资源放极端安全，小部分放极端冒险，中间地带最危险。你等不到完整信息，但可以设计不对称的收益结构。

反脆弱是第二个框架从认知层到操作层的延伸。

说白了就是：能算清楚的算清楚，算不清楚的别装能算清楚。前者靠脑力，后者靠心力。

到这里，整套框架才真正丰满。

然后根据这套策略，我衍生出了 5 个 Skill，每个 Skill 对应一个大师级人物，形成了一套组合拳。为了好记，我给每个 Skill 起了一个中文名字，命名为**太虚五转**。

| 转 | Skill | 大师 | 中文名 | 干什么 |
|:--:|-------|------|--------|--------|
| 一 | osborn | Alex Osborn | 散怀 | 发散思维，把可能性空间铺开 |
| 二 | hegel | 黑格尔 | 澄源 | 迭代收敛，把发散的东西收拢 |
| 三 | bayes | 贝叶斯 | 叩实 | 事实验证，拿真实数据打脸 |
| 四 | feynman | 费曼 | 破妄 | 输出审查，找自欺的点 |
| 五 | debono | 德·博诺 | 归真 | 价值回溯，捡回被误杀的 |

这五个 Skill 串在一起，就是一条完整的认知链路：**发散 → 收敛 → 验证 → 审查 → 归真**。

卡老师说的对抗式审查，就对应我的第四转，费曼破妄。但前面三转和后面一转，同样重要。

下面我一个一个讲。

---

## 一转·散怀（Osborn）：先把空间打开

**对应大师：Alex Osborn，头脑风暴之父**

面对一个问题，脑子里第一个冒出来的想法，往往会把后面的可能性全挤掉。这叫**收敛过早**。

我自己在做投资分析的时候就经常这样。比如看到某个标的不太好，第一反应就是「这公司不行」，然后整个分析就沿着这个方向走，再也想不到别的角度。

第一转 Osborn 解决的就是这个问题。奥斯本有句原话：把一个野想法收一收，比凭空想出一个新想法容易。

所以它做的事情很简单：先不评判、不筛选、不排序，量先到位。然后用 SCAMPER 七算子（替代、组合、改造、修改、它用、去除、反转）做结构化补盲，再用利益相关方视角轮换（用户视角、技术视角、竞争者视角、极端视角）扫一遍剩下的盲区。

举个真实案例。

前段时间我研究 Anthropic 官方发布的金融仓库（32K+ stars），想看看有什么可以借鉴的。如果按以前的习惯，我大概会直接列几个觉得有用的点，然后开始落地。

但这次我先跑了一轮 Osborn 发散。

结果一发不可收拾，产出了 144 个想法。其中有个方向特别关键：方案对自身现状的描述可能不准确。这个信号在后续验证中被证实：方案声称「Nightly Review 偏自由格式」，实际上代码里已有五层结构化框架；声称「台账追溯需要架构重构」，实际已有完整的 `_lineage` 对象。

如果不是发散阶段把这些「前提假设」摆出来，后面直接按方案执行，就会在已经解决的问题上白费力气。

144 个想法，最后聚类为 12 条核心断言，进入下一转。

## 二转·澄源（Hegel）：浊水静置，杂质沉淀

**对应大师：黑格尔，辩证法**

发散完了，144 个想法摊在那儿。但哪些是真问题？哪些是脑子一热写出来的？哪些互相矛盾？

第二转 Hegel 做的是迭代收敛。名字来自黑格尔的辩证法，正题、反题、合题。但不是空谈，它落地是一条工程管线。

具体做法：把多种分析方法串成自动循环管线，每轮走三个环节：校准、挑战、深化，直到不确定项稳定下来。

- **校准**对齐事实：这个断言有证据吗？置信度多少？
- **挑战**引入对立视角：主动攻击自己的结论，找偏差。
- **深化**把存活的结论往下挖一层：根因是什么？边界在哪？

每轮结束检查收敛条件：置信度稳定了吗？有没有新发现？有没有翻转？稳定了就停，没稳定就再来一轮。

还是上面那个 Anthropic 的案例。

144 个想法进入 Hegel 后，聚类为 12 条核心断言，按严重度排序。其中最严重的一条是：「借鉴方向反了」，置信度 0.55，跟抛硬币差不多。

Hegel 还有一个很关键的设计：**零假设检验**。不是只找问题，还要问「如果原方案没问题，我们应该看到什么证据？」这个设计后来是因为 Feynman 审查发现了一个结构性偏差才加上的，后面会讲到。

## 三转·叩实（Bayes）：用事实打脸，而不是用逻辑自圆其说

**对应大师：贝叶斯，概率论之父**

到这里，12 条断言已经有了置信度。但置信度是 AI 估的，能信吗？

第三转 Bayes 干的事，用贝叶斯定理的大白话讲：你有了一个判断（先验），现在拿真实数据去检验它，看看到底要不要修正你的判断（后验）。

具体做法是：把每条结论拆成具体的、可查证的预测，「如果结论成立，那么 A/B/C 应该是这样」，然后拿真实数据去验。查 API 返回、读代码实现、跑 shell 命令、调两个不同的 AI 模型交叉对质。预测被数据打脸的地方，就是你的认知盲区。打脸越狠，认知盈余越大。

为什么要调两个不同的 AI 模型？因为同一个模型族共享训练数据，可能有共同盲区。Claude 觉得对的东西，换个模型未必同意。两个不同模型独立解读同一段代码，结论一致才算数，不一致本身就是信号。

费曼说「你不能欺骗自己」，Bayes 这一步最容易被自欺绊住。但验不验得了，靠的是数据，不是觉悟。

继续 Anthropic 的案例。

12 条断言拆出了 14 条预测，三路并行 Explore agent 查代码、查文档、查配置。

结果，35% 的预测被打脸了。

最狠的两个打脸：

| 方案描述 | 实际代码状态 | 后果 |
|---------|------------|------|
| 「Nightly Review 偏自由格式」 | 已有五层结构化框架（五类路由 + 四步财报分解 + 三态 FC + 凭证门禁 + 20+ 字段） | 第 1 号借鉴点从 P0 降为搁置 |
| 「台账追溯链需要架构重构」 | 已有完整 `_lineage` 对象（source_row + source_line + parser_version + timestamp + fingerprint） | 第 3 号借鉴点从 P1 降为微增强 |

还有一条：方案声称本项目 skill 平均 3-8KB，有「膨胀风险」。实际测量：平均 12.9KB，最大 83.3KB。膨胀已经发生了，而且是项目自身演化的结果，不是借鉴导致的。

如果没有 Bayes 拿真实代码去验，我大概率会花几天时间去「重构」一个已经有 `_lineage` 的台账追溯链。

## 四转·破妄（Feynman）：你不能欺骗自己

**对应大师：费曼，物理学家**

这一转，就是卡老师说的**对抗式审查**。

但 Feynman 比简单的对抗式审查多了一层：BUG 好找，自欺难防。

AI 有一个倾向：自圆其说。它跟你说「这样对」，不会说「可能有问题」。人也一样，写完就不想推翻自己。

Feynman 用的六条原则，借鉴了卡帕西 Claude.md 里的几条核心原则，但我把它从写代码扩展到了更广泛的领域。

| 原则 | 防的自欺类型 | 什么意思 |
|------|------------|-----------|
| P1 停下来想 | 假设未验证 | 「我觉得是这样」，你觉得不算数 |
| P2 不确定性诚实 | 伪装确定性 | 「这个值肯定是 X」，你把猜测写成了事实 |
| P3 设计要简 | 过度设计 | 「万一以后需要呢」，你在为不存在的需求写代码 |
| P4 执行要准 | 附带伤害 | 「顺手改一下」，你在制造无法追溯的变更 |
| P5 验收要闭环 | 自证清白 | 「应该没问题」，你在用想象替代实验 |
| P6 数据要溯源 | 真相漂移 | 「这里也写一份」，你在制造终将偏离的副本 |

执行方式上有个设计：审查员是独立的 subAgent，只看变更产物，看不到对话历史。

隔离的是上下文，不是能力。审查者拿到的是 diff + 原则 + checklist，没有你的推理过程，没有用户的原始需求，没有「为什么这样改」的任何解释。它只能从变更本身判断是否自洽。同一个 Agent 写的代码自己审，上下文全是「为什么这样做」的理由，你已经在为自己辩护了。AI 自圆其说的能力太强了，强到你自己都信了。把理由拿掉，代码还站得住，那才算真站得住。

Anthropic 案例的 Feynman 阶段，发现了流程级的结构性偏差。

独立 subAgent 对 Hegel + Bayes 的结论链做审查，最重要的发现在流程层面，跟单条断言的对错无关：

F1：为了找问题而找问题。12 条断言全是「问题」，0 条「确认合理」。修正方向全是降级，0 条升级。这意味着：无论方案是否真有问题，分析结果都会是「发现了很多问题」。

F2：严重度方法论缺陷。severity = impact alone，不乘 confidence。最严重的一条断言（critical）置信度只有 0.55，比抛硬币好不了多少。这是「最坏情况思维把不确定猜测包装成紧急问题」。

F3：前提未验证。12 条断言中有 4 条的前提未经验证就获得了严重度标签。

这个发现催生了三个改进项：Hegel 增加零假设检验类别、严重度必须耦合置信度（conf < 0.6 不得标 critical）、Bayes 每个断言同时拆零假设预测。

费曼审查也找代码 BUG，但它更在意你思考过程中的 BUG。专门的代码 bug 审查，我用 ce-code-review 这种专业 skill。

## 五转·归真（Debono）：在批判的废墟里捡回不该被埋没的东西

**对应大师：爱德华·德·博诺，水平思维之父**

这一转最容易被忽略，也最有意思。

前四转形成了一条强大的批判管线：发散 → 收敛 → 验证 → 审查。但批判管线有盲区，它擅长排除，不擅长保留。

批判有时下手太重，把有价值的东西连同不可靠的部分一起扔掉了。

Debono 补的就是这块盲区：给被否定的命题做逆向价值扫描。不问「为什么错」，问「在什么条件下对」。

它有五种构建策略：

- **S1 逆向利益相关方**：找到被批判视角忽略的群体，分析他们为何从中受益
- **S2 条件缩窄法**：逐步缩窄适用条件，找到「窄但真实」的成立区间
- **S3 时间延迟价值**：分析未来什么条件变化会让价值涌现
- **S4 类比复活法**：在其他领域找到类似命题成功的案例
- **S5 反面代价法**：如果不采纳，会失去什么机会

Anthropic 案例的归真阶段，5 个被否的命题全部找到了存活条件。

比如，方案第 1 号借鉴点「财报结构化」被 Bayes 打脸（因为已经有五层框架了），但 Debono 发现：现有框架聚焦会计质量，缺竞争定位变化和资本配置效率两个维度。不替换，增量补充。

方案第 2 号「数据源四层协议」被缩窄为三层，但 Debono 发现：去掉空架子后，核心价值是冲突解决规则（当 Futu 和 akshare 返回不同价格时以谁为准），这个规则当前完全缺失。

最终结论：原方案 6 个借鉴点，经过五转打磨，全部被条件缩窄为增量改进，没有一个照搬。

搭完这个框架之后，我在跟 AI 聊天时发现它跟德·博诺的「六顶思考帽」暗合。前四转分别对应绿帽（发散）、蓝帽（收敛）、白帽（事实）、黑帽（批判），第五转正好补上黄帽（价值肯定）。但有一顶我没做成 Skill——红帽。红帽代表直觉和情感，我认为这是人独有的东西，不该交给大模型。所以在 Debono 这一步留了个口子，允许人把自己的直觉判断反馈进去，补上最后这顶帽子。

## 五转连招：taixu-pipeline

五个 Skill 可以单独用，也可以串成连招。

我写了一个编排器 `taixu-pipeline`，一键串联 osborn → hegel → bayes → feynman → debono，保证完整五转流程不被降级。

每一步之间有 CHECKPOINT，你可以选择继续、跳过、或终止。也可以 `--steps hegel,bayes` 只跑其中几步。

还有一个防嵌套机制：Hegel 管线内部本身就会调用 Bayes 和 Feynman，如果在 pipeline 里再跑一遍，会导致嵌套降级。所以 pipeline 调用 Hegel 时会传入 `--standalone` 参数，跳过内嵌调用，由 pipeline 后续步骤用独立 Skill 的完整能力来跑。

**Anthropic 借鉴方案的完整五转过程：**

| 转 | Skill | 做了什么 | 核心数字 |
|:--:|-------|---------|---------|
| 一 | Osborn | 144 个想法发散 | P1: 85, P2a: 38, P2b: 21 |
| 二 | Hegel | 12 条断言收敛 + 9 条 Requirements | confirmed: 1, uncertain: 1, dismissed: 3 |
| 三 | Bayes | 14 条预测验证 | 35% 被打脸, 7 条确认 |
| 四 | Feynman | 6 原则审查 | 3 个流程级偏差, 净效应系统性降低信心 |
| 五 | Debono | 5 个存活论证 | 6 个借鉴点全部条件缩窄为增量改进 |

五转之前，方案的目标是「cherry-pick Anthropic 的设计模式」。五转之后，结论变了：外部参照的价值在「暴露盲区」，不在「拿来用」。它暴露了 3 个内部视角看不到的改进点，同时确认了本项目 4 项 Anthropic 不具备的独特创新。

## 外围 Skill：苏格拉底诘问（Socrates）

太虚五转管的是「自己的推理和产出」。但投资分析里，你每天都会碰到大量外部观点：看空报告、卖方分析、媒体报道、雪球帖子。

费曼能审这些吗？不能。费曼防的是自欺，审的是自己写的东西。外部观点的问题不是你会自欺，是别人可能在忽悠你。

Socrates 就是干这个的。它审的是外部论点，用苏格拉底诘问法的三个维度：

- **事实核查**：论点中的事实声明准确吗？数据有没有选择性引用？案例描述有没有失真？外部论点最常见的手法不是撒谎，是挑着引用。
- **对称性检查**：这个批评是否同样适用于对照组？如果批评对双方都适用但只攻击一方，那就是论证技巧。攻击一方的弱点叫分析，攻击双方的共性叫修辞。
- **独立性检查**：去掉这条论点，其他论点还成立吗？如果一条被否定会连带否定其他，整条推理链就是脆弱的。看似五六条论据，其实拴在一根绳上。

费曼和苏格拉底，防的是两种不同方向的认知漏洞。

费曼防自欺：你写的东西自己审，脑子里全是「为什么这样做」的理由，你已经在替自己辩护了。费曼用独立 subAgent 把这些理由隔离掉，让审查者只看产物本身。

苏格拉底防被忽悠：别人说得头头是道，数据、图表、逻辑链都有，你听完觉得「确实有道理」。但他引用的数据准不准？他的批评标准对对照组适用不适用？他这几条论点是不是一根绳上的蚂蚱，抽掉一根全盘崩塌？

一个往内看，找自己脑子里的漏洞；一个往外看，找别人论证里的漏洞。

| 维度 | Feynman 费曼审查 | Socrates 苏格拉底诘问 |
|------|-----------------|---------------------|
| 审查对象 | 自己的推理/产出 | 外部论点/观点 |
| 方向 | 内省 | 外审 |
| 核心问题 | 我哪里可能自欺？ | 对手的论证可靠吗？ |
| 方法论 | 六条自欺原则 | 三维度审查（事实+对称性+独立性） |
| 执行方式 | 独立 subAgent，隔离上下文 | 调用 bayes 做事实核查 + 对称性/独立性分析 |
| 使用场景 | 写完代码/方案后、提交前 | 看到看空报告/第三方分析后 |

举个例子。前段时间康方生物有个看空观点：「HARMONi-6 仅中国患者，全球适用性存疑」。

Socrates 跑了三维度审查：

1. 事实核查：HARMONi-6 确实是中国试验，但 HARMONi（全球 III 期）有 38% 非中国患者，且 BLA 已获 FDA 受理。**事实部分错误**。
2. 对称性检查：看空方批评 HARMONi-6 人群特征（90% 男性、排除 75+ 岁），但 K 药对照组同样受限于相同入组标准。这是**选择性攻击，不是独立弱点**。
3. 独立性检查：「仅中国数据 → FDA 不会批准 → 商业化失败」的推理链，第一步就被事实打脸。**链式脆弱**。

综合评估：质量评级「低」，建议「排除」。这个看空观点不纳入决策参考。

外部论点最常见的问题很少是「完全错误」，更多是「部分准确 + 论证技巧 + 链式脆弱」。说得似是而非，你感觉哪里不对但说不上来。Socrates 就是帮你把这个直觉拆成三个维度，逐一判定。

Socrates 不在太虚五转的连招管线里，因为它的触发条件不同：五转是你在产出时主动跑的，Socrates 是你看到外部观点时按需触发的。两者互补，一个管内部质量，一个管外部输入。

## 不止是 Vibe Coding

和卡老师说的一样，这两个 Prompt 的应用范围远不止代码。

太虚五转也是。

**做投资分析的时候**，先 Osborn 发散所有可能的角度，Hegel 收敛出核心判断，Bayes 拿真实数据验，Feynman 审查自己有没有在骗自己，最后 Debono 看看被否的标的是不是真的没价值。

**做架构决策的时候**，Osborn 铺开所有可能的方案，Hegel 迭代收敛找盲区，Bayes 查代码验证前提是否成立，Feynman 找流程级偏差，Debono 捡回被过度批判的设计。

**甚至做人生决策的时候**，先发散所有可能性，再收敛出核心选项，用事实验证每个选项的前提，审查自己的思考过程有没有偏差，最后看看被否掉的选项里有没有被误杀的。

因为太虚五转处理的是认知本身的问题，跟具体领域无关。

发散管空间，收敛管方向，验证管事实，审查管自欺，归真管价值。

## 最后

卡老师说，第一性原理和对抗式审查，是他心目中 Vibe Coding 的两大基石。

我的太虚五转，是在第一性原理的基础上，往前加了发散和收敛，往后加了归真。费曼审查就是对抗式审查，但多了六条具体原则和独立 subAgent 隔离机制。

这六个 Skill 我已经开源到 GitHub，欢迎大家体验。每个 Skill 都可以单独使用，也可以用 taixu-pipeline 一键连招。

说实话，这套方法论不是一天想出来的。它是在我做了大半年的投资分析 AI 系统过程中，从实际痛点里长出来的。每一次用，都会发现流程本身的偏差，然后迭代改进。上面 Anthropic 案例里 Feynman 发现的「为了找问题而找问题」的结构性偏差，就是最好的例子。如果没有这套审查机制，这个偏差永远发现不了。

方法论这东西，光精巧没用，得能被代码执行、被机器强制。

这也是我把压箱底的方法论做成 Skill 的原因。光写篇文章讲讲没意义，得让这套流程嵌进 AI 每次分析的过程里：先发散够再收敛，收敛完拿数据验，验完找人审，审完还得回头看。哪一步偷懒，流程本身就会报错。

人类靠自觉守纪律，AI 靠机制守纪律。

我相信机制。
